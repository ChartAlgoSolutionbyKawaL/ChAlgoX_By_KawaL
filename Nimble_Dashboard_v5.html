How to Read This Dashboard

Abbreviations, scoring logic, and a weekly review workflow — static reference, not data-driven

🔤 Abbreviations & Terms

BUBusiness Unit — the internal department a project serves.
% DoneTask-level completion percentage, entered/maintained per task.
Avg ProgressMean of % Done across all tasks in a project — not weighted by task size/duration.
Risk LevelManually tagged High/Medium/Low — qualitative judgment of impact if the task slips.
PriorityHigh/Medium/Low business importance of the task itself.
At RiskNot yet overdue, but trending toward a missed date or showing warning signs.
DroppedPermanently removed from scope — this task/project will not be delivered. Read the Notes column for the reason why.
OverdueEnd Date has passed and Status is not Completed (computed daily against Today).
Health (R/A/G)Computed project rollup — see scoring logic below, not a manual tag.

🧮 Scoring & Health Logic

Project Health (Mgmt Summary scorecard):
🔴 RED — more than 2 issue tasks (Delayed+At Risk+Dropped)
🟡 AMBER — 1–2 issue tasks, OR zero issues but avg progress < 30%
🟢 GREEN — zero issue tasks and avg progress ≥ 30%

Overall Status badge: On Track ≥75% avg completion · At Risk 40–74% · Delayed <40%.

Risk Exposure Matrix: cross-tab of Risk Level × Priority, counting only incomplete tasks — top-right (High/High) is the cell to watch first.

Action Items (every page): rule-based and recomputed on every filter change — ranked by a severity score (overdue days > dropped > high-risk/high-priority > ownership concentration > backlog), never hardcoded text.

🛡️ 3-Layer Data Validation System

How the dashboard checks your Excel file before loading — what it catches, what it skips, and what to fix
🗂️
Layer 1 — Input Mapping
Detects which columns are present and correctly named. Checks that the required sheet exists. Flags any required column that is missing — if a required column is absent, you cannot proceed to Layer 2.
⛔ Missing required column = upload blocked
⚙️
Layer 2 — Rule Engine
Runs 20 exception rules across all rows — checking completeness, value validity, date logic, and business rules. Each rule is classified as Error (row excluded), Warning (row included, noted), or Info (awareness only).
⚠️ Errors exclude rows · Warnings are noted
🚦
Layer 3 — Output Gate
Final render decision. Shows exactly how many rows are ready, excluded, and warned. If any rows are excluded, a persistent Data Quality Notice banner appears on every dashboard page so users know the analysis scope.
✅ Clean · ⚠️ Proceed with warnings · 🛑 Fix & re-upload
Rule Reference — All 20 Checks
Rule Category What it checks Severity Fix in Excel
R01CompletenessAny required field (Project, Business, Phase, Activity, Owner, Status, Priority, Risk, Dates, %Done) is blankErrorFill in all mandatory columns for the row
R02CompletenessOwner column is empty — no accountability for the taskErrorEnter at least one owner; use Name+Name for co-owners
R03CompletenessStatus is Delayed or At Risk but both Remark and Notes are blankWarnAdd root cause in Remark column so meeting reviewers understand why
R04CompletenessStatus is Dropped but no explanation in Remark or Notes — drop rationale undocumentedErrorAdd drop reason to Remark column before saving (mandatory)
R05CompletenessActivity / Task name cell is emptyErrorEvery row needs a task name — blank rows are excluded
R06Value ValidityStatus is not one of the 6 permitted valuesErrorUse exactly: Completed / On Track / At Risk / Delayed / Dropped / Not Started
R07Value ValidityPriority value is not High / Medium / LowErrorUse a dropdown in Excel to prevent typos
R08Value ValidityRisk Level is not High / Medium / LowErrorUse a dropdown in Excel to prevent typos
R09Value Validity% Complete is not a number between 0.0 and 1.0ErrorEnter as decimal (0.75 = 75%) or percentage (75%) — both are accepted
R10Value ValidityStatus is Completed but % Done is less than 100%WarnSet % Done to 1.0 (100%) for Completed tasks
R11Date LogicStart Date or End Date is missing — Gantt bar cannot be plottedErrorAll tasks need both dates. Use YYYY-MM-DD or Excel date format
R12Date LogicEnd Date is earlier than Start Date — impossible rangeErrorCorrect the dates; check for copy-paste errors between columns
R13Date LogicDuration column differs from End−Start by more than 2 daysWarnUpdate Duration to match the date range, or correct one of the dates
R14Date LogicStatus is Completed but Start Date is in the futureWarnCheck if the start date was entered incorrectly
R15Date LogicNon-Completed task whose End Date is more than 90 days in the pastInfoConfirm whether the task is still active or should be closed/dropped
R16Business LogicSame Activity name appears twice in the same Phase of the same ProjectWarnRename one task to distinguish it — duplicates produce double bars on Gantt
R17Business LogicStatus is Not Started but End Date is already in the pastWarnUpdate status to Delayed, or extend the end date to the new expected delivery
R18Business LogicHigh Priority task has no owner assignedErrorEvery High Priority task must have a named owner before uploading
R19Business LogicStatus is At Risk but % Complete is 100% — contradictory stateWarnChange Status to Completed if the task is done
R20Business LogicSame owner has two active (non-Completed, non-Dropped) tasks whose date ranges overlap by more than 5 daysInfoReview owner capacity — not an error, but worth flagging in the weekly meeting
Error Row excluded from all analysis — fix and re-upload for complete data
Warn Row included — issue noted in Data Quality banner on each page
Info Row included — awareness only, no banner
📋 Before Every Upload — Quick Checklist for Data Authors
✅ Sheet is named "All Tasks Detail"
✅ Status only uses the 6 permitted values
✅ All Delayed / At Risk tasks have a Remark
✅ All Dropped tasks have a Remark explaining why
✅ Every row has Start Date ≤ End Date
✅ % Done is 1.0 for every Completed task
✅ Every High Priority task has an Owner
✅ No duplicate Activity names within the same Phase
